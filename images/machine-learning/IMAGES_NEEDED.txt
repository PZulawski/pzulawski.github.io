GPT ADVENTURES — PHASE 4  (place in /images/gpt-adventures/)

  phase4-bucketing.png
    -> Gradient bucketing visualization — CUDA timeline showing AllReduce patterns
       for different bucket sizes (1MB, 25MB, larger). Bottom row is a separate
       CUDA stream showing overlapped collective ops.

  phase4-topology.png
    -> GPU topology diagram — 8xA100 slice showing two subgroups of 4 GPUs,
       PCIe connections within subgroups and DCI host bridge between subgroups.

  phase4-allreduce-timeline.png
    -> CUDA timeline showing AllReduce operations and their overlap (or lack
       thereof) with backward compute — illustrating the gap before the
       optimiser step.

  phase4-scaling-pcie.png
    -> Training performance comparison across device counts on PCIe — showing
       that adding GPUs becomes counterproductive due to communication overhead.

  phase4-nvswitch-topology.png
    -> NVSwitch topology diagram — direct GPU-to-GPU connections via dedicated
       NVSwitch switching hardware.

  phase4-nvswitch-hgx.png
    -> HGX A100 NVSwitch configuration diagram (borrowed from Nvidia Developer
       page) — showing 12 NVLink connections per GPU across 6 NVSwitch boxes.

  phase4-scaling-nvswitch.png
    -> Training speed vs. number of devices with NVSwitch — sustained speed-up
       even at 8 GPUs, gains becoming marginal beyond that.

  phase4-allreduce-latency.png
    -> AllReduce latency comparison between the 40GB and 80GB A100 GPU boxes
       — showing different effective bandwidths despite identical NVLink specs.

  phase4-scaling-efficiency.png
    -> Near-linear scaling efficiency with NVSwitch — AllReduces largely
       overlapped with compute up to 8 GPUs.

  phase4-ddp-vs-fsdp.png
    -> DDP vs FSDP performance comparison — showing marginal difference for
       our model size.

  phase4-memory-checkpointing.png
    -> Allocated GPU memory percentage — with and without gradient checkpointing.

  phase4-throughput-checkpointing.png
    -> Throughput (tok/s) — with and without gradient checkpointing, showing
       the re-computation overhead.
